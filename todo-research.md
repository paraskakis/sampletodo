# Todo API Market Research

## ICPs
1. **Individual users** who want a simple way to manage their personal tasks and to-do lists.

## Needs & Pain Points
* Keep track of tasks across multiple projects without losing context
* Quickly capture a new task before forgetting it — low friction is essential
* See what's due today or overdue without manually sorting through everything
* Mark tasks as done and feel the progress — completed items should be easy to filter out
* Organize tasks into projects so work, personal, and side-project items don't bleed together

## Benefits

### Individual Users
* Stay on top of commitments — nothing falls through the cracks
* Feel in control of their day by seeing a clear, prioritized list of what to do next
* Reduce mental load by offloading task tracking to a system they trust

## Use Cases

Based on common task management patterns, in order of priority:

1. List all tasks, optionally filtered by project or completion status
2. Create a new task with a title, optional description, due date, and priority
3. Get details for a specific task
4. Mark a task as complete (or reopen it)
5. Update a task's title, description, due date, or priority
6. Delete a task
7. List all projects
8. Create a new project
9. List tasks within a specific project
10. Delete a project
11. Update a projects's name and description

**Note:** The API requires authentication — each user manages their own tasks and projects.
